# Memory Crystal 架构详解

## 系统架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    Memory Crystal 架构                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   ┌──────────────┐     ┌──────────────┐                         │
│   │  fact_store   │────▶│ fact_feedback │                        │
│   │  (结构化存储)  │◀────│  (反馈进化)   │                        │
│   └──────┬───────┘     └──────────────┘                         │
│          │                                                       │
│          ▼                                                       │
│   ┌──────────────────────────────────────────────────────────┐  │
│   │                    Crystal Facets (切面)                   │  │
│   ├──────────────────────────────────────────────────────────┤  │
│   │  ◆ add      - 注入新晶体（添加事实）                       │  │
│   │  ◆ search   - 关键词折射（搜索）                          │  │
│   │  ◆ probe    - 实体全息回忆（查询某实体的所有事实）         │  │
│   │  ◆ related  - 关联光线追踪（查找关联实体）                │  │
│   │  ◆ reason   - 多面交叉推理（跨实体组合推理）              │  │
│   │  ◆ contract - 裂痕检测（发现矛盾）                        │  │
│   │  ◆ update   - 晶体重塑（更新事实）                        │  │
│   │  ◆ remove   - 切面消除（删除事实）                        │  │
│   │  ◆ list     - 全景扫描（列出所有事实）                    │  │
│   └──────────────────────────────────────────────────────────┘  │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 数据模型

每条事实（Crystal Facet）包含以下字段：

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `fact_id` | int | 唯一标识符 | 41 |
| `content` | str | 事实内容 | "MiMo 输入>28K tokens 时响应变慢" |
| `category` | str | 类别 | general / user_pref / project / tool |
| `tags` | str | 标签（逗号分隔） | "mimo,performance,bottleneck" |
| `trust_score` | float | 信任分数 (0.0-1.0) | 0.5 |
| `retrieval_count` | int | 被检索次数 | 5 |
| `helpful_count` | int | 被标记有用次数 | 2 |
| `created_at` | str | 创建时间 | "2026-06-08 10:30:00" |
| `updated_at` | str | 更新时间 | "2026-06-08 10:30:00" |

## 类别系统

| 类别 | 用途 | 示例 |
|------|------|------|
| `user_pref` | 用户偏好/习惯 | "用户对自动化失败容忍度低" |
| `project` | 项目相关 | "Memory Crystal 命名于 2026-06-08" |
| `tool` | 工具/技术 | "MiMo 输入>28K tokens 时响应变慢" |
| `general` | 通用知识 | "WSL 代理端口 10808" |

## 信任分数机制

```
初始值: 0.5
    │
    ├── helpful (+0.1) ──▶ 最高 1.0
    │
    └── unhelpful (-0.1) ──▶ 最低 0.0
```

**影响**：
- 高信任事实优先展示
- 低信任（<0.3）可定期清理
- 高频检索间接提升信任

## 检索机制

### 1. 关键词搜索 (search)
- 全文匹配
- 支持多关键词（AND 逻辑）
- 适合模糊查询

### 2. 实体探查 (probe)
- 精确匹配实体名称
- 返回该实体的所有事实
- 适合深度查询

### 3. 关联追踪 (related)
- 查找与实体关联的其他实体
- 基于事实中的实体共现
- 适合发现隐藏关联

### 4. 跨实体推理 (reason)
- 多实体交集查询
- 返回同时涉及多个实体的事实
- 适合复杂推理

### 5. 矛盾检测 (contract)
- 检查事实间的逻辑冲突
- 基于内容相似度和语义对比
- 适合知识验证

## 设计理念

### 晶体隐喻

1. **切面 (Facet)**：每条事实是晶体的一个切面
2. **折射 (Refraction)**：信息通过关联产生新洞察
3. **裂痕 (Fracture)**：矛盾是晶体内部的裂痕
4. **光泽 (Luster)**：信任分数决定事实的"光泽度"
5. **结晶 (Crystallization)**：知识从混沌到有序的过程

### 核心原则

1. **结构化**：不只是文本，而是有类别、标签、信任的知识
2. **可推理**：支持跨实体查询和矛盾检测
3. **可进化**：通过反馈机制持续优化
4. **可验证**：信任分数反映知识的可靠性
5. **可扩展**：未来可支持更多实体类型和关系
